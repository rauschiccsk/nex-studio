## Key decisions

- **Embedded terminal nahrádza WT taby — single page per role** (single-session, project picker pri spawn vyhradil sa krok 2). Quality-first: 1 sidebar entry = 1 role = 1 mental model. Dual-Build Audit (Tibor's test) ostáva sekvenčne v 1 Implementer session, lebo robí sa zriedka (per release).

- **Container/host bridge — Variant A** (claude CLI v BE image + volume mount). BE container už drží JWT secret + DB credentials cez `.env` — pridanie claude auth do toho istého trust boundary je konzistentné. Alternatívy (host-runner daemon, per-session container) sú over-engineering pre internal NEX Studio context.

- **Single-session-per-role-per-user dvojito enforced** — in-memory pre-check (rýchly 409) + DB partial unique index `WHERE ended_at IS NULL` (concurrency safety). Belt-and-suspenders pattern.

- **WS auth via query-string token** — browser WebSocket API nedovolí custom headers. JWT decode identický s REST `Authorization: Bearer` → žiadna duplicita validation kódu.

- **Output ring buffer in-memory** (BE proces) — 512 chunks × 4KB ≈ 2MB per session. Persistuje cez refresh klienta (re-attach replays), NIE cez BE reštart. Trade-off akceptovaný — persistent buffer cez reštart by vyžadoval externý PTY supervisor (tmux/disk), over-engineering pre v1.

- **Idle TTL na `last_input_at`, nie `last_output_at`** — dlhé claude generácie sú legitímne dlhé sessions, output stream ich nemá udržiavať alive umelo. Dvojhodinová "thinking" generácia neprolonguje 24h lease.

- **Selected Project ako globálny stav nahrádza per-page picker** — Director's UX insight: viacero stránok bude potrebovať "vybraný projekt", duplikovať selektor v každej je friction. `activeContextStore` rozšírený o 2 nezávislé sloty (`selectedProject` + `selectedVersion`) — pin v `/projects` je explicitná akcia, version sa auto-syncuje pri navigácii do VersionDetail.

- **Disabled state > hidden state** pre context-závislé linky — Director's directive "Workflow nemá ukazovať projekt": Workflow link je viditeľný ale neklikateľný keď chýba verzia. Discoverability ostáva, ale link nikdy nevedie na zlú content doménu. Tooltip navedie k Versions ako single legitímnej ceste.

- **Auto-pick verzia pri pin** — riešenie mismatch medzi "verzia existuje v DB" a "frontend store ju nepozná". Pri pin fetch versions → priorita `status="active"` → fallback prvá v zozname (`version_number DESC`) → ak žiadna, Workflow ostane legitímne disabled. Optimistic UI eliminuje vizuálnu pauzu pri pin.

- **Tri iterácie Workflow fix** — môj pôvodný design predpokladal, že fallback Workflow `/projects` je akceptovateľný. Director ma 3× korigoval: (1) Workflow nemá ísť na Projects, (2) Workflow nemá ísť na Versions (duplikát), (3) Workflow nemá byť disabled keď verzia existuje. Lesson: Director's UX princíp "links lead to their content domain, never fall back to a different domain" je nekompromisný — disabled state je preferovaný pred krížením domén.
