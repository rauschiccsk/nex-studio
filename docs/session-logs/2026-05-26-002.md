### 2. NEX Inbox bug fix — IMAP UIDVALIDITY parse (commit `23e46e8`)

**Diagnostika:** Po reštarte backendu IMAP poller každý cyklus zlyhával s `IMAP server did not expose UIDVALIDITY` warning + NIB-050 unavailable threshold exceeded (~45 min). Faktúra v Stalwart mailboxe sa nepullovala.

**Root cause:** `backend/apps/email_poller/imap_client.py:129` predpokladalo `getattr(client, "uidvalidity", None)` po `client.select()`. `aioimaplib.IMAP4` túto property **NEEXPONUJE** (`dir(IMAP4)` len `['uid', 'uid_search']`; `grep uidvalidity` v aioimaplib zdrojáku → 0 hits). Server (Stalwart) odpoveď RFC-compliant: `* OK [UIDVALIDITY 2005102545] UIDs valid` — client ju len ignoroval. Porušený BE BEHAVIOR §5.1 idempotency invariant (composite key `(imap_uid_validity, imap_uid)` bez validity = broken).

**Fix:** Delegovaný AG Implementer-ovi NEX Inbox cez NEX Studio AgentTerminal (paste-on-screen workflow — moja pôvodná navigácia k `nex-implementer` CLI wrapperu bola zlá, Director ma opravil). Implementer commit `23e46e8`: regex parse `\[UIDVALIDITY (\d+)\]` zo SELECT response data, store v `ImapSession.uidvalidity` field, +125/-9 LOC, 8 nových unit testov, 557 BE tests green. Lokálne nasadené v UAT cez `docker compose build` z `/opt/projects/nex-inbox/` (NEX Inbox nemá Git remote ani CI).

**Verifikácia po deploy:** logy ukazujú `imap_poller.uidvalidity_changed: old=0, new=2005102545` + `imap_poller.fetched_uids: count=1` — parser funguje, email stiahnutý, žiadne ImapError. Reálna PDF (204 KB) + email.eml (286 KB) uložené v `/var/lib/inbox/originals/4816400b-acce-4fd7-8cce-fab4ebe27532/`.

## Key decisions

- **SMB service account namiesto admin C$ share** — least-privilege (NTFS ACL len Modify, nie SeBackup/Restore), audit-friendly. NTFS owner súborov potvrdený ako `nex-inbox-svc`, nie Administrator.
- **CIFS mount na host úrovni, nie v kontajneri** — Docker volume bind mount priekopne `/mnt/mager-edocs-inbox` z host fstab. Kontajner nepotrebuje cifs-utils, mount survive-uje kontajner recreate.
- **NEX Inbox bug fix cez AG Implementer NEX Studio** — namiesto direct edit; audit trail + DONE report + Tibor test cesta. Paste-on-screen workflow pretože NEX Studio delegation mechanizmus (Koordinátor F-001 pickup z `delegations` table) ešte nie je hotový.
- **NEX Inbox local-only repo akceptované pre dnes** — žiadny Git remote ani CI workflow. UAT rebuild lokálny cez `docker compose build`. Flag pre samostatnú diskusiu (porušuje ICC štandard pre projekty po Phase D).
