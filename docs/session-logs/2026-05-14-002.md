## Key decisions

- **L-016 fix two-tier**: konkrétny projekt sed-replace + template-level `{{PROJECT_SLUG}}` substitution. Bez template fixu by každý budúci ICC projekt narazil na ten istý blokér. Plus L-016 LESSONS_LEARNED entry pre future audit visibility. Quality-first: opravujeme symptóm aj koreň naraz.

- **OQ review patterns**: Designer's krok-za-krokom OQ-flow s explicit "schvaľujem alebo navrhnem korekciu" je vysoko produktívny — moja zóna ako second-opinion validátor (cez WebFetch pre external specs, cez ICC_STANDARDS pre tech stack) zachytila Peppol EAS error pred zápisom. Director's role: minimal touch (1× word `Schvaľujem` per OQ + Director-level UX corrections); moja role: deep tech verification (EAS codes, Schematron pinning, library compliance, banned libs check).

- **WebFetch pre external spec verification** — pri Peppol BIS 3.0 mapping table som odporúčal Designerovi verify cez WebFetch aktuálny EAS Code List (v9.6, November 2025). Designer to urobil a našiel rozpor (9914 = AT, nie SK). To je proven pattern: pri external standard references nie spoľahnúť sa na pamäť (môj knowledge cutoff January 2026, EAS list je živý dokument), vždy WebFetch. Pridáva ~30s per OQ ale eliminuje rejected Peppol gateway dokumenty post-release.

- **Post-Gate-D handoff = moja zóna**: Designer charter §2 ho zakazuje commitnúť icc-knowledge ZBKB súbory (governance scope mimo Designer allow-list). Plus Qdrant offline počas Designer's session znamená že RAG reindex zlyhal silently. Moja role pri handoff: (a) verify services online, (b) commit KB súbory + push, (c) explicit reindex pre files ktoré obišli Write/Edit auto-hook (e.g. cez `cp -r`). Bez tohto by Implementer pri spawn dostal stale RAG search results.

- **customer-requirements per-version pattern** — Director's structural observation odhalila že `customer-requirements.md` mal byť per-version od začiatku, nie ako top-level. Rovnaký pattern ako `CHANGES.md`. Pre v0.2.0+ Director napíše nový súbor pod `docs/specs/versions/v0.2.0/`. Template fix eliminuje opakovanie chyby pri budúcich projektoch.

- **Filesystem-realistic view > curated subset** — Project Specs pôvodne filtroval LEN `.md` ako "spec browser" pattern. Director's mental model je iný: chce vidieť **reálny obsah disku** (import/, export/, akýkoľvek typ súboru). Quality-first reinterpretácia: browser tools by mali zrkadliť ground truth (filesystem), nie kurátorský subset. Edit semantics zostali .md-only (Director scope: typo fixes na spec dokumentoch).

- **DISABLE_AUTOUPDATER namiesto custom claude install** — alternative bolo prebuildnúť image bez ~/.claude mount alebo manuálne nakopírovať claude CLI do containera. Variant A (mount + auto-update silence) je čistejšia separation: hostiteľ má svoju claude install, container má svoju (cez npm install), auth state share-ovaný cez mount. Update lifecycle deterministicky cez Docker rebuild, žiadny runtime fetch.
