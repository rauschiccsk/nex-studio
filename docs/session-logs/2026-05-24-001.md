## Workflow disciplína + lessons (validated dnes)

### Comprehensive review pattern validated

Sub-agent design audit (3 paralelné, ~22 min) našiel 14 findings ktoré by inak vyžadovali ~12-18 h iterative smoke test cycles. Jeden Designer round 2 + jeden Implementer round 3 vs odhadovaných **5-7 bug-fix cycle round trips**.

**Lesson:** Po prvom reálnom deploy FAIL, comprehensive review je **vždy** lacnejší ako reactive bug-by-bug fix discovery. Per quality-first 4 kritérií.

### Designer self-audit pred commit (per memory `feedback_designer_self_audit`)

CR-022 amendment ~280 LOC vyžadovala sub-agent self-audit pred commit → zachytené 5 vlastných chýb (numbering cascade B-1, stale refs B-2, count inconsistency A-1, naming inconsistency A-4/C-1, scope wording A-2). Bez self-audit by skončili v commit-e a Implementer by ich riešil iteratívne.

### Variant menu anti-pattern

Director flagol dvakrát dnes "ako spraviť aby si neponúkal varianty bez zmyslu" (smoke test plan). Memory `feedback_quality_first` existuje — issue je **aplikačná disciplína** , nie nové pravidlá. Mám NOPASSWD sudo a mal som execute, nie delegovať na Direktora.

### DB_CONNECTION_VARS whitelist (Dedo Q4 drobnosť)

Dedo flagol pattern matching ako risk (false positive `PRODUCTION_DB_BACKUP_URL`) → explicit whitelist `{DATABASE_URL, DB_HOST, DB_PORT, DB_NAME, DB_USER, DB_PASSWORD, POSTGRES_*}` namiesto regex. Implementer aplikoval per request. **Lesson:** "rozumný pattern" často zachytáva edge cases ktoré DROP guard preven Variants.
