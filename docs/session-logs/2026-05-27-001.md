### 4. Threshold downgrade journey + reality check

Director's pragmatic cieľ "PDF + XML v SMB cez normal pipeline" zablokovaný **systematic AI confidence pod spec routing thresholds**:

- gemma4 8B na 2-page invoice: aggregate **0.762** confidence (decision "manual" lebo < 0.80 MEDIUM)
- gemma3:27b na 2-page: aggregate **0.668** (predtým, ešte nižšie)
- 14-page CANIS PDF: 55 min Ollama na CPU, NIB-034 invalid JSON drift
- 4 downgrade attempts: HIGH 0.95→0.70, MEDIUM 0.80→0.70, CRITICAL_AUTO 0.90→0.50, CRITICAL_MIN 0.85→0.50

**Director's reality check**: *"Aký má to zmysel?"* — postupný downgrade demontuje spec safety design. Pri 0.50 critical threshold prejdú aj invoices s nízkou IČO/dátum/suma istotou bez review = zbytočne risky.

Reálna diagnostika: spec defaults sú pre **GPU + top-tier modely** (Llama 70B+, Mistral Large) kde confidence systematicky 90-99%. CPU + small models (gemma4 8B / gemma3:27b Q4) **nikdy** nedosiahne 95%+ HIGH threshold. Threshold downgrade = fingerpainting nad fundamentálnym mismatch deployment ↔ spec.
