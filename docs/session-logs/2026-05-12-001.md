## Key decisions

- **Three specialized agents** (Designer/Implementer/Auditor) — architektonicky vynútená izolácia rolí cez per-agent CLAUDE.md + settings.json. Možnosť C (per-agent session) zvolená pre quality-first.
- **Universal CLAUDE.md ako base, per-agent appendovaný cez `--append-system-prompt`** — single source of truth pre univerzálne pravidlá, žiadna duplikácia.
- **§2 Vývojová metodológia — WATERFALL ako záväzné ICC pravidlo** — explicitné odmietnutie agile, dôraz na plánovanie, automatizácia implementácie cez agentov.
- **Tiborov test (Dual-Build Audit)** — validácia kvality špecifikácie cez nedeterminizmus implementácie. MANDATORY pre release.
- **Doc tree: live + `versions/vX.Y.Z/spec/` + `CHANGES.md`** — live = stav po poslednej released; per-verzia spec = plné sekcie self-contained pre Implementera; CHANGES = stakeholder changelog (immutable po release).
- **`customer-requirements.md` aditívny** — Zoltán píše ručne; Designer ho LEN číta.
- **Auto-create v0.1.0 planned pri Create Project** — Designer's Step 0 VERSION binding nájde verziu bez manuálneho setupu.
- **ARCHITECT.md plná deprecation** — per-agent session logs (`docs/session-logs/<role>/`) ho granulárnejšie nahrádzajú. Existujúce súbory ostávajú ako historický artefakt.
- **Domain variant → Designer charter** (nie do hlavného) — doménový kontext je relevantný pre návrh, nie pre kód/audit.
- **ICC-wide dáta v KB** (Team, Port Registry, Approved Projects, Tech Stack) — single source of truth, v CLAUDE.md len odkazy.
